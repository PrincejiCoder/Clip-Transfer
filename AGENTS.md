**AGENTS**
- **Run server**: `cargo run --release` (default bind 0.0.0.0:8080). Override with env/CLI: `MICROBIN_BIND`, `MICROBIN_PORT`, `--bind`, `--port`.
- **Docker**: `docker build -t linkdrop . && docker run -d -p 8080:8080 linkdrop`. No extra steps needed; image copies built binary.
- **Tests**: `cargo test --verbose`. All unit/integration tests run.
- **Data storage**: defaults to `microbin_data` directory (`MICROBIN_DATA_DIR`). Contains SQLite DB (or JSON if `MICROBIN_JSON_DB=true`). Ensure directory is writable.
- **Routing**:
  - `/` – home page.
  - `/{slug}` – view existing paste, edit mode (if `allow_edit`), or create page when slug missing.
  - `/{slug}?created=true` – share UI with QR code.
  - `/raw/{slug}` – plain‑text raw content.
  - Reserved slugs (`static`, `public`, `favicon.ico`, `raw`, `api`, `admin`). Cannot be used for pastas.
- **Slug validation**: regex `^[a-z0-9-_]{3,50}$`. Normalized (lower‑case, spaces→`-`, trimmed non‑alphanum).
- **Env/CLI flags** (most common):
  - `MICROBIN_BASIC_AUTH_USERNAME` / `..._PASSWORD` – HTTP basic auth.
  - `MICROBIN_ADMIN_USERNAME` / `..._PASSWORD` – admin UI.
  - `MICROBIN_PUBLIC_PATH` – URL prefix when behind reverse proxy.
  - `MICROBIN_HIDE_HEADER`, `MICROBIN_HIDE_FOOTER`, `MICROBIN_HIDE_LOGO` – UI toggles.
  - `MICROBIN_ENABLE_BURN_AFTER`, `MICROBIN_DEFAULT_BURN_AFTER` – burn‑after‑read support.
  - `MICROBIN_DEFAULT_EXPIRY`, `MICROBIN_MAX_EXPIRY` – expiry presets.
  - `MICROBIN_DISABLE_TELEMETRY` – disables telemetry thread.
  - `MICROBIN_THREADS` – worker count for Actix.
  - `MICROBIN_GC_DAYS` – days after which unused pastas are pruned.
  - `MICROBIN_QR` – enable QR code generation (requires `MICROBIN_PUBLIC_PATH`).
  - `MICROBIN_CUSTOM_CSS` – URL to custom stylesheet.
- **Feature flags** (via env):
  - `MICROBIN_JSON_DB` – use JSON DB instead of SQLite.
  - `MICROBIN_NO_FILE_UPLOAD` – disable file uploads.
  - `MICROBIN_READONLY` – server‑wide upload lock; `MICROBIN_UPLOADER_PASSWORD` required to create.
  - `MICROBIN_ENABLE_READONLY` – per‑pasta read‑only option in UI.
  - `MICROBIN_ENCRYPTION_CLIENT_SIDE` / `MICROBIN_ENCRYPTION_SERVER_SIDE` – client‑side / server‑side encryption.
- **Telemetry**: started automatically unless `--disable-telemetry`/`MICROBIN_DISABLE_TELEMETRY`.
- **Cleanup**: background thread removes expired pastas every minute.
- **CLI usage**: All flags are generated by `clap`; `--help` shows full list.
- **Important gotchas**:
  - Slug must not conflict with reserved list; otherwise `403`.
  - Large payloads (>1 MiB) are rejected (`413`).
  - When `public_path` is set, generated share URLs use it; otherwise fallback to `http://localhost:PORT`.
  - Changing a flag after data is created does not retroactively affect existing pastas (e.g., encryption, privacy).
- **Common dev workflow**:
  1. `cargo build --release`
  2. `cargo run --release` (or Docker)
  3. Open `http://localhost:8080`.
  4. Run `cargo test` before committing.
- **CI**: GitHub Actions run `cargo build` then `cargo test` on push/PR.
